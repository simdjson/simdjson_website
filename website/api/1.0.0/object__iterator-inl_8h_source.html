42 //
43 // ### Live States
44 //
45 // While iterating or looking up values, depth >= iter.depth. at_start may vary. Error is
46 // always SUCCESS:
47 //
48 // - Start: This is the state when the object is first found and the iterator is just past the {.
49 // In this state, at_start == true.
50 // - Next: After we hand a scalar value to the user, or an array/object which they then fully
51 // iterate over, the iterator is at the , or } before the next value. In this state,
52 // depth == iter.depth, at_start == false, and error == SUCCESS.
53 // - Unfinished Business: When we hand an array/object to the user which they do not fully
54 // iterate over, we need to finish that iteration by skipping child values until we reach the
55 // Next state. In this state, depth > iter.depth, at_start == false, and error == SUCCESS.
56 //
57 // ## Error States
58 //
59 // In error states, we will yield exactly one more value before stopping. iter.depth == depth
60 // and at_start is always false. We decrement after yielding the error, moving to the Finished
61 // state.
62 //
63 // - Chained Error: When the object iterator is part of an error chain--for example, in
64 // `for (auto tweet : doc["tweets"])`, where the tweet field may be missing or not be an
65 // object--we yield that error in the loop, exactly once. In this state, error != SUCCESS and
66 // iter.depth == depth, and at_start == false. We decrement depth when we yield the error.
67 // - Missing Comma Error: When the iterator ++ method discovers there is no comma between fields,
68 // we flag that as an error and treat it exactly the same as a Chained Error. In this state,
69 // error == TAPE_ERROR, iter.depth == depth, and at_start == false.
70 //
71 // Errors that occur while reading a field to give to the user (such as when the key is not a
72 // string or the field is missing a colon) are yielded immediately. Depth is then decremented,
73 // moving to the Finished state without transitioning through an Error state at all.
74 //
75 // ## Terminal State
76 //
77 // The terminal state has iter.depth < depth. at_start is always false.
78 //
79 // - Finished: When we have reached a }, we are finished. We signal this by decrementing depth.
80 // In this state, iter.depth < depth, at_start == false, and error == SUCCESS.
81 //
